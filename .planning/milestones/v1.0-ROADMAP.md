# Milestone v1.0: Commerce & Accounts

**Status:** ✅ SHIPPED 2026-03-29  
**Phases:** 1-4  
**Total Plans:** 10

## Overview

Shipped a full authenticated commerce experience: persistent Firebase Auth, Razorpay checkout with verified totals, order visibility for users and admins, admin image uploads via Firebase Storage, and storefront filtering/search.

## Phases

### Phase 1: Auth & Persistence

**Goal**: Securely adapt the Next.js boundary to handle Firebase Auth persistently across pages, enabling the creation of Wishlists and pre-saved Shipping Addresses attached to a UID.  
**Depends on**: –  
**Plans**: 1 plan

Plans:

- [x] 01-01: Auth & Persistence

**Details:** Auth flows persist via cookies across SSR; wishlists and addresses stored per UID; logout form wired; build passes with SSR auth.

### Phase 2: Razorpay Checkout Flow

**Goal**: Bridge the native `Bag` to a secure payments model using Razorpay API, verifying totals actively via the server rather than trusting the React array.  
**Depends on**: –  
**Plans**: 3 plans

Plans:

- [x] 02-01: Cart Scoping & Currency Migration
- [x] 02-02: Checkout & Razorpay Integration
- [x] 02-03: Order Confirmation

**Details:** Cart and checkout forced dynamic for cookie access; server actions recompute totals and stage pending orders; HMAC verification; confirmation page renders paid orders.

### Phase 3: Order Management

**Goal**: Surface the `orders` collection efficiently to both the end-user (for their receipts) and the Admin (for fulfillment).  
**Depends on**: –

### Phase 4: Catalog Extensibility

**Goal**: Overhaul catalog management by removing remote text-based `https://` URLs for images using Firebase Storage instead, alongside building dynamic storefront URL Param search features.  
**Depends on**: –

## Milestone Summary

**Key Decisions:**
- Force dynamic rendering for cart, checkout, and confirmation routes to allow session-cookie access.
- Recompute cart totals server-side before payment; Razorpay HMAC verification required for capture.
- Adopt Firebase Storage for admin image uploads to keep infrastructure consistent.
- Use URL params as single source of truth for storefront filtering/search.

**Issues Resolved:**
- Static prerender failures on cookie-dependent routes resolved by dynamic rendering flags.
- INR currency formatting standardized across cart, checkout, orders, and admin views.

**Issues Deferred:**
- None recorded.

**Technical Debt Incurred:**
- Nyquist validation docs/tests pending for phases 2–4; Razorpay sandbox end-to-end run still recommended.
